# WireGuard Helpers

Набор утилит для управления WireGuard и отслеживания расходования трафика.

## Структура

- **wg_usage.py** — Ежедневный отчёт об использовании WireGuard, отправляемый в Telegram
- **pywg/** — Telegram бот для управления конфигами WireGuard через WG-Easy API

## wg_usage.py

Скрипт автоматически читает счётчики трафика с WireGuard, группирует их по пользователям и отправляет дневной отчёт в Telegram.

### Функции

- ✅ Дневной отчёт по расходу трафика за последние 24 часа
- ✅ Еженедельный отчёт по расходу в месяц
- ✅ Уведомление о переходе в следующий месяц (отчёт за прошлый)
- ✅ Оповещение при превышении установленного лимита (80%, 90%, 100%)

### Установка

```bash
# Установите переменные окружения
export TELEGRAM_BOT_TOKEN="ваш_bot_token"
export TELEGRAM_CHAT_ID="ваш_chat_id"
export WG_CONTAINER="wg-easy"  # имя контейнера
export WG_INTERFACE="wg0"      # интерфейс
```

### Использование

Запустить вручную:
```bash
python3 wg_usage.py
```

**Рекомендуется** запускать раз в день через cron (например в 10:00):
```bash
0 10 * * * /путь/к/wg_usage.py
```

### Конфигурация

Переменные окружения:
- `TELEGRAM_BOT_TOKEN` — Token бота от @BotFather
- `TELEGRAM_CHAT_ID` — ID чата, куда отправлять отчёты
- `WG_CONTAINER` — Имя контейнера с wg-easy (по умолчанию `wg-easy`)
- `WG_INTERFACE` — Имя WireGuard интерфейса (по умолчанию `wg0`)
- `WG_JSON_PATH` —路径 к wg0.json внутри контейнера
- `WG_USAGE_STATE` — Путь к файлу состояния (по умолчанию `/root/wg_usage_state.json`)
- `VPN_LIMIT_BYTES` — Месячный лимит трафика в байтах (по умолчанию 1 ТБ)
- `ALERT_THRESHOLDS` — Пороги уведомлений в процентах (по умолчанию `80,90,100`)
- `WG_REPORT_TZ` — Временная зона (по умолчанию `Asia/Tomsk`)

## pywg/ — Telegram Bot

Бот для управления WireGuard конфигами через WG-Easy REST API. Пользователи могут запрашивать и создавать конфиги прямо из Telegram.

### Функции

- 📥 Получить готовый конфиг (в виде файла и QR-кода)
- 🆕 Создать новый конфиг с автоматической нумерацией
- 👀 Просмотреть расход трафика всеми пользователями
- 🔐 Контроль доступа по whitelist (allowed_users.txt)
- 🚀 Неограниченные конфиги для VIP пользователей

### Установка

#### 1. Скопируй .env.example и заполни значения

```bash
cd pywg
cp .env.example .env
# Редактируй .env и добавь свои credentials
```

#### 2. Создай файлы с разрешением

```bash
# Скопируй пример и отредактируй
cp allowed_users.txt.example allowed_users.txt
cp unlimited_users.txt.example unlimited_users.txt
```

#### 3. Запуск через Docker Compose

```bash
docker-compose up -d
```

Или локально (требует Python 3.12+ и зависимости):
```bash
pip install -r requirements.txt
python3 bot_api.py
```

### Команды бота

| Команда | Описание |
|---------|---------|
| `/start` | Открыть главное меню (кнопки для выбора конфига) |
| `/get N` | Получить конфиг номер N (например `/get 2`) |
| `/new` | Создать новый конфиг |
| `/usage` | Показать расход трафика по всем клиентам |
| `/allow @username` | Добавить пользователя в allowed (только для админа) |
| `/help` | Справка |

### Кнопки интерфейса

- **#1, #2, #3...** — Получить готовый конфиг
- **➕ Новый (#N)** — Создать следующий конфиг

### Структура allowed_users.txt

```
username1
@username2
# комментарии начинаются с #
```

Имена без `@` будут автоматически преобразованы в `@username`.

### Конфигурация

**Переменные .env:**

```bash
BOT_TOKEN=123456:ABC-DEF...              # Token от @BotFather
WG_PASSWORD=YourWebPassword             # Пароль WG-Easy вебинтерфейса  
WG_API_BASE=http://127.0.0.1:51821/api  # URL WG-Easy API
MAX_CONFIGS_PER_USER=4                  # Лимит конфигов на обычного пользователя
```

## Развёртывание

### Требования

- Python 3.12+ (если запускать локально) или Docker
- WireGuard с WG-Easy
- Telegram бот (создать через @BotFather)

### Быстрый старт

1. Клонируй репозиторий
2. Заполни .env с чувствительными данными
3. Запусти:
   ```bash
   docker-compose up -d
   ```

### Безопасность

⚠️ **Никогда не коммитьте .env файлы с реальными токенами!**

- `.env` добавлен в `.gitignore`
- Используй `allowed_users.txt` для контроля доступа
- Используй environment variables вместо hardcode
